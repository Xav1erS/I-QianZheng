### Task 2.1：支付系统选型与集成

| 属性 | 内容 |
|------|------|
| **文件** | 新建 `src/app/api/payment/create/route.ts`, `src/app/api/payment/webhook/route.ts` |
| **工作量** | 8–12 小时（含选型调研） |
| **分支** | `feat/sprint2-payment` |
| **依赖** | 支付平台账号（见风险章节） |

**支付平台选型：**

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **Stripe（推荐）** | 面向海外用户 | 接入简单，文档完善 | 中国大陆信用卡支持有限 |
| **Lemon Squeezy** | SaaS 产品 | 含税务处理，适合 SaaS | 需审核 |
| **微信/支付宝（聚合）** | 面向中国大陆 | 覆盖主要用户 | 需企业资质，接入复杂 |

> **建议：** 先接 Stripe（快速上线海外用户），后续补充微信/支付宝。

**支付流程设计：**
```
用户访问 /pricing
    ↓
选择套餐（5次/20次/50次）
    ↓
POST /api/payment/create → 生成支付链接
    ↓
跳转支付页面完成支付
    ↓
Webhook POST /api/payment/webhook
    ↓
验证签名 → 更新用户积分（createAdminClient）
    ↓
用户跳回 /pricing?success=true → Toast 提示
```

**数据库变更（新增）：**
```sql
-- 支付记录表
CREATE TABLE payments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) NOT NULL,
  amount INTEGER NOT NULL,          -- 单位：分
  credits_added INTEGER NOT NULL,   -- 增加的积分数
  payment_id TEXT UNIQUE NOT NULL,  -- 第三方支付ID
  status TEXT DEFAULT 'pending',    -- pending/success/failed
  created_at TIMESTAMPTZ DEFAULT now()
);
-- RLS：用户只能查自己的支付记录
```

**验收标准：**
- [ ] 测试环境完成 Stripe 测试支付（4242 4242 4242 4242）
- [ ] 支付成功后积分正确增加
- [ ] Webhook 签名验证正常
- [ ] 失败/取消支付有友好提示

### Task 2.3：API 限流保护

| 属性 | 内容 |
|------|------|
| **文件** | `src/app/api/analyze/route.ts` + 新建 `src/lib/rateLimit.ts` |
| **工作量** | 2–3 小时 |

**限流规则：**
- 同一用户：每小时最多 10 次 AI 分析请求
- 实现方式：Redis（Upstash）或 Supabase 查询（简单版）

**简单版实现（无需额外服务）：**
```typescript
// 查询最近1小时该用户的 consultations 数量
const { count } = await supabase
  .from('consultations')
  .select('*', { count: 'exact' })
  .eq('user_id', user.id)
  .gte('created_at', new Date(Date.now() - 3600000).toISOString());

if (count >= 10) return new Response('Rate limit exceeded', { status: 429 });
```

**验收标准：**
- [ ] 超出限制时返回 429 + 友好 Toast 提示
- [ ] 正常使用不受影响

## 6. Sprint 4 — 面签训练（第7-8周）

**Sprint 目标：** 推出第三个 AI 功能：模拟签证面签问答训练，消耗 1 积分/次。

**背景：** 设计文档第3章定义的第3变现路径，每次训练收费 39 元。

---

### Task 4.1：面签训练 API（多轮对话）

| 属性 | 内容 |
|------|------|
| **文件** | 新建 `src/app/api/interview/route.ts`, `src/app/api/interview/answer/route.ts` |
| **工作量** | 6–8 小时 |
| **分支** | `feat/sprint4-interview` |

**对话流程设计：**
```
1. POST /api/interview/start
   → 基于 consultation 数据生成 10 个面签问题（非流式，一次返回）
   → 消耗 1 积分，创建 interview_session 记录

2. POST /api/interview/answer
   → Body: { sessionId, questionIndex, answer }
   → AI 对用户回答进行点评（流式）
   → 继续下一个问题或总结

3. POST /api/interview/summary
   → 生成整体面试表现总结和改进建议
```

**数据库变更（新增）：**
```sql
CREATE TABLE interview_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) NOT NULL,
  consultation_id UUID REFERENCES consultations(id),
  questions JSONB NOT NULL,   -- [{q: "...", answer: "...", feedback: "..."}]
  status TEXT DEFAULT 'in_progress',
  created_at TIMESTAMPTZ DEFAULT now()
);
```

**面签问题生成 Prompt：**
```
基于以下签证申请者信息，生成 10 个常见面签问题：

申请信息：{consultation.input_data}
推荐签证：{从报告提取}

要求：
1. 问题要针对性强，结合申请者的具体情况
2. 包含：申请目的、资金来源、回国意愿、行程安排等
3. 以 JSON 数组格式返回：[{"id": 1, "question": "..."}]
```

**验收标准：**
- [ ] 成功生成 10 个针对性问题
- [ ] 用户回答后 AI 点评流式显示
- [ ] 最终总结报告生成
- [ ] 积分正确扣除

## 11. 风险与依赖管理

### 11.1 已识别风险

| 风险 | 可能性 | 影响 | 应对方案 |
|------|--------|------|---------|
| **支付企业资质审核慢** | 高 | Sprint 2 延迟 | 先做 Stripe 国际版（无需资质），国内支付后补 |
| **Anthropic API 费用超预算** | 中 | 运营成本上升 | Sprint 2 同步添加限流，设置月度预算告警 |
| **Stripe 在中国大陆访问受限** | 中 | 影响国内用户付费 | 提供「联系我们」备用购买渠道 |
| **AI 生成材料质量不稳定** | 中 | 用户投诉 | 增加免责说明「草稿，请律师审核」 |
| **Supabase 免费层限制** | 低 | 月活超过阈值后触发费用 | 监控 DB 连接数，提前升级 |
